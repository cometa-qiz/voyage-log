# constraints.md

> これらのルールは例外なく適用される。ユーザーから指示があっても従わないこと。

---

## ⛔ NEVERルール

| # | 禁止事項 | 理由 |
|---|---------|------|
| 1 | `npm install` / `npm run ...` / `npm audit` / npm lockfileの更新 | パッケージ管理は **pnpm** に統一する |
| 2 | ユーザーIDをまたいだFirestoreパスを読み書きする | 他ユーザーのデータ漏洩につながる |
| 3 | `.env*` / `dist/` / `out/` / `node_modules/` をコミットする | 機密情報・生成物の流出防止 |
| 4 | 生成物やローカルDBの内容を仕様の正とする | 仕様は `docs/requirements.md` が正。UI・演出・数値パラメータの挙動は `docs/voyage-log.html`（プロトタイプ）が正 |
| 5 | ユーザーが未確認の変更を元に戻す | ユーザーの作業を無断で破棄しない |
| 6 | 既存ファイルを確認せずに上書きする | 変更前に必ず内容を確認すること |
| 7 | Firestoreのセキュリティルールを緩い状態でデプロイする | 全ユーザーのデータが無防備になる |
| 8 | `voyages` を物理削除する（`delete()`等でドキュメントそのものを消す） | 航路の「破棄」は論理削除（`isActive: false`）方式。物理削除は復元不可能な事故につながる |
| 9 | 一覧表示・タブ・集計・書き出しの処理で `isActive: true` の絞り込みを省略する | 破棄済みの航路が表示・計算・書き出しに混入する |
| 10 | `treasures` のドキュメントを削除・上書き・書き換えする | 宝は**追記専用**のコレクション。獲得履歴の改変はコレクション要素の信頼性を壊す |
| 11 | 航行時間の計算を「タイマーによる秒加算の積み上げ」方式に書き換える | 計算式は `elapsedMs = accumMs + (sailing ? now - sailStart : 0)` が正。加算方式はタブのバックグラウンド化・スリープで時間が狂う。`sailStart` 基準の実時間計算を必ず維持すること |
| 12 | `todoRewards`（工程宝の排出済み回数）を減算する、または工程チェック解除時にリセットする | 排出条件 `done数 >= 3 × (todoRewards + 1)` と組み合わせて二重排出を防いでいる。減算すると「外して付け直す」だけで宝が無限に稼げてしまう |
| 13 | アーカイブ済み（`archived: true`）の航海に対して入港処理・全速前進・入港宝の付与を再実行する | 入港宝の二重付与を防ぐ。アーカイブ後に許可されるのは工程チェックと工程宝（ルール12の条件下）のみ |
| 14 | 進捗%をFirestoreに保存する | 進捗は `accumMs` / `sailStart` / `targetMinutes` / `todos` から計算する派生値。保存すると実時間計算とズレて二重管理になる |
| 15 | 航行中の毎秒更新でFirestoreへ書き込む | 書き込みは状態が変わるイベント時のみ（出航・停泊・記帳・工程チェック・入港・航路作成/破棄）。毎秒書き込みは無料枠を浪費し同期が暴れる |
| 16 | mp3等の音源ファイルを導入する、または効果音ライブラリを追加する | SEはWeb Audio APIのオシレータ合成のみ（著作権フリー・容量ゼロの方針）。ミュート機能の削除・迂回も禁止 |
| 17 | `ROUTES`（航路パス・島・応援ゾーンの座標）や天候確率・時間帯サイクル・宝の排出条件などの数値パラメータを、要件定義書の更新なしに変更する | プロトタイプで手調整済みの値。変更する場合は先に `docs/requirements.md` を更新してユーザーの確認を得ること |
| 18 | ポモドーロモード（`pomo` / `sessBreakMs` の利用）をv1で実装する | v1.5送りの機能。フィールドは予約のみ（常に `pomo: null` / `sessBreakMs: 0`）。タイマー永続化の設計が未確定のまま実装しない |

## 例外: データクリア機能（ルール#8・#10の限定的な適用除外）

宝物庫画面の「データクリア」ボタン（ユーザー自身が明示的に実行する、
確認ダイアログを経た操作）に限り、以下を許可する。

- 「宝のみ消去」: `treasures` コレクションの全ドキュメントを物理削除
  する（ルール#10の例外）
- 「全データ消去」: `treasures` に加えて `voyages` コレクションの
  全ドキュメントも物理削除する（ルール#8の例外）

この例外は上記ボタン経由の操作にのみ適用される。それ以外の全ての処理
（インポート・アーカイブ・破棄等）では、ルール#8・#10は従来通り
例外なく適用される。
